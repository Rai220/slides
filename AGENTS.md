Здесь собраны презентации созданные с использованием slidev
Используй скилл slidev для работы с презентациями
Если пользователю нужен редактируемый `pptx`, а не набор картинок, используй скилл `slidev-editable-pptx`
Для запуска используй bunx slidev

## Как работать с картинками во всех презентациях

- **Локальные картинки конкретной презентации**: клади рядом со `slides.md` в папку презентации и подключай как `src="image.png"` или `![](image.png)` (без `./`).
- **Общие ресурсы на несколько презентаций**: клади в корневую папку `public/` и подключай как `src="/path/in/public.png"` или `![](/path/in/public.png)`.
  - Это работает при наличии `vite.config.ts` в папке презентации с `publicDir: '../public'` (см. ниже).
- **Фоны/обложки**: для background в frontmatter используй путь из `public/`, например `background: /bg.png`.
- **QR-коды**: по умолчанию создавай через URL сервиса `https://api.qrserver.com/v1/create-qr-code/?size=220x220&data=<urlencoded-data>` и вставляй как обычную картинку в `src`. Если QR нужен как локальный ассет, сохраняй его рядом с `slides.md` для одной презентации или в `public/` для повторного использования.

При создании новой презентации всегда добавляй в её папку файл `vite.config.ts` с содержимым:
```ts
import { defineConfig } from 'vite'

export default defineConfig({
  publicDir: '../public',
})
```
Это необходимо для доступа к общим ресурсам из папки `public/` (фото, QR-коды и т.д.).

Для выгрузки в PDF используй:
```bash
bunx slidev export <путь к md> --output <путь к pdf> --timeout 60000 --per-slide
```
Флаг `--per-slide` обязателен - без него фон может обрезаться на последних слайдах.

Для редактируемого `pptx` не используй `bunx slidev export --format pptx`, потому что Slidev экспортирует такие слайды картинками. Вместо этого делай отдельный `slides_pandoc.md`, адаптируй разметку под Pandoc и экспортируй через:
```bash
pandoc <путь к slides_pandoc.md> -t pptx -o <путь к editable.pptx>
```

## Форматирование текста

- **Никогда не используй длинные тире (—)**. Всегда используй обычный дефис-минус `-`.
