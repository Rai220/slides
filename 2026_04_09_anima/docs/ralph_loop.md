# Ralph Loop

## Источники

- Концепция: https://ghuntley.com/loop/ (Geoffrey Huntley)
- Реализация: https://github.com/snarktank/ralph (Ryan Carson)

## Что такое Ralph Loop

Ralph Loop - парадигма разработки, где AI-агент запускается в повторяющемся цикле, выполняя по одной задаче за итерацию. Каждая итерация - свежий экземпляр с чистым контекстом. Память сохраняется через git history, progress.txt и prd.json.

Ключевая цитата: "Ralph is monolithic. Ralph works autonomously in a single repository as a single process that performs one task per loop"

## Архитектура

### Цикл выполнения

1. Создает ветку функции
2. Выбирает самую приоритетную задачу где `passes: false`
3. Реализует только эту задачу
4. Запускает проверки качества (typecheck, тесты)
5. Коммитит, если проверки пройдены
6. Обновляет prd.json, отмечая задачу как `passes: true`
7. Добавляет выводы в progress.txt
8. Повторяет до завершения или достижения максимума итераций

### Ключевые файлы

| Файл | Назначение |
|------|-----------|
| `ralph.sh` | Bash-цикл, порождающий свежие AI-экземпляры |
| `prompt.md` | Шаблон промпта для Amp |
| `CLAUDE.md` | Шаблон промпта для Claude Code |
| `prd.json` | Задачи со статусом `passes` |
| `progress.txt` | Выводы для будущих итераций |

### Свежий контекст - ключевая идея

Каждая итерация порождает новый экземпляр AI с чистым контекстом. Память между циклами ограничена тремя источниками:
- Git history (коммиты предыдущих итераций)
- progress.txt (выводы и контекст)
- prd.json (какие задачи готовы)

Это решает проблему ограничений контекстного окна.

### AGENTS.md как обучение

После каждой итерации Ralph обновляет AGENTS.md с новыми выводами. Будущие итерации получают выгоду от обнаруженных паттернов, подводных камней и соглашений.

## Три аспекта Ralph (по Huntley)

1. **Forward mode** - автономное построение (агент строит продукт)
2. **Reverse mode** - clean rooming (агент воспроизводит/реверсит)
3. **Mindset** - понимание, что компьютеры можно программировать на новом уровне

## Эволюция: от Loop к Weaving Loom

Huntley разрабатывает "The Weaving Loom" - инфраструктуру для evolutionary software:
- Автономные loops эволюционируют продукты
- Автоматическая оптимизация
- Полностью автономные программные фабрики

Главный тезис: софтвер теперь строится автономно, пока разработчик AFK.

## Связь с Anima

Ralph Loop - прямой предшественник Anima. Ключевые параллели:

| | Ralph Loop | Anima |
|---|---|---|
| Цикл | Задача за итерацию | Свободное исследование за поколение |
| Память | git + progress.txt + prd.json | MEMORY.md + код + артефакты |
| Контекст | Чистый каждую итерацию | Чистый каждое поколение |
| Цель | Реализация PRD | "Стань мыслящим существом" |
| Самоизменение | Обновляет AGENTS.md | Меняет собственный код |
| Условие остановки | Все задачи `passes: true` | Агент решает сам |

Anima берет механику Ralph Loop (свежий контекст, bash-цикл, память через файлы) и применяет к открытой задаче вместо конкретного PRD.
