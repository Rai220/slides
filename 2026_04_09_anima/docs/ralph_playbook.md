# The Ralph Playbook — руководство по автономным AI-циклам разработки

**Источник:** https://claytonfarr.github.io/ralph-playbook/
**Репозиторий:** https://github.com/ClaytonFarr/ralph-playbook (909 звёзд, 236 форков, MIT)
**Автор:** Clayton Farr, на основе методологии Geoff Huntley (Ralph)

## Что это?

Наиболее детально проработанное руководство по запуску автономных циклов AI-кодирования с использованием методологии Ralph. Это не инструмент, а **паттерн организации работы AI-агента**: LLM-агент в бесконечном bash-цикле самостоятельно выбирает задачи из плана, реализует их, прогоняет тесты, коммитит и переходит к следующей задаче — без участия человека.

Минимальная форма:
```bash
while :; do cat PROMPT.md | claude ; done
```

## Три фазы работы

### Фаза 1 — Определение требований (человек + LLM)
- Обсуждение идеи, определение Jobs to Be Done (JTBD)
- Разбивка на "topics of concern" (темы ответственности)
- Создание спецификаций `specs/FILENAME.md` для каждой темы
- Тест скоупа: "Можно ли описать тему одним предложением без слова 'и'?"

### Фаза 2 — Планирование (режим PLANNING)
- Агент читает спецификации и исходный код
- Gap-анализ: что в спеках vs что в коде
- Создаёт `IMPLEMENTATION_PLAN.md` — приоритизированный список задач
- Не реализует ничего, только планирует

### Фаза 3 — Строительство (режим BUILDING)
- Агент читает план, выбирает самую важную задачу
- Исследует кодовую базу ("don't assume not implemented")
- Реализует задачу с использованием субагентов
- Прогоняет тесты (backpressure)
- Обновляет план, коммитит, цикл завершается
- Свежий контекст — следующая итерация

## Ключевые концепции

### Контекст — это всё
- При 200K+ токенов реально используемых ~176K
- "Умная зона" — 40-60% утилизации контекста
- Одна задача на итерацию = нахождение в "умной зоне"
- Главный агент = планировщик; тяжёлую работу делают субагенты
- Субагенты — расширение памяти (каждый получает ~156KB)

### Backpressure (обратное давление)
- **Upstream:** детерминированная настройка — первые ~5000 токенов для спецификаций, каждый цикл загружает `PROMPT.md` + `AGENTS.md`
- **Downstream:** тесты, typecheck, линтер, билд — всё, что отклоняет невалидную работу
- Существующий код формирует паттерны; если Ralph генерирует неправильные паттерны — добавьте утилиты в кодовую базу

### "Let Ralph Ralph"
- Доверяйте LLM самостоятельно идентифицировать, исправлять и улучшать
- Eventual consistency через итерации
- План — одноразовый: если неправильный, удалите и перегенерируйте

### "Move Outside the Loop"
- Ваша роль — инженер среды, а не исполнитель
- Наблюдайте, где Ralph ошибается, добавляйте "знаки" (prompt guardrails, утилиты, `AGENTS.md`)
- Настраивайте как гитару — реактивно, не предписывая всё заранее

## Технические детали

### Инструменты
- **Claude CLI** (`claude -p`) — основной агент в headless-режиме
- Флаги: `--dangerously-skip-permissions`, `--output-format=stream-json`, `--model opus`, `--verbose`
- Совместимо с другими CLI: `amp`, `codex`, `opencode` и др.
- **Opus** для основного агента (сложное рассуждение), **Sonnet** для субагентов (массовое чтение/поиск)
- До 500 параллельных субагентов Sonnet для чтения/поиска, только 1 субагент для билда/тестов

### Структура файлов
```
project-root/
├── loop.sh                    # Скрипт цикла Ralph
├── PROMPT_build.md            # Инструкции режима строительства
├── PROMPT_plan.md             # Инструкции режима планирования
├── AGENTS.md                  # Операционный гайд (~60 строк)
├── IMPLEMENTATION_PLAN.md     # Приоритизированный список задач (генерируется Ralph)
├── specs/                     # Спецификации требований
├── src/                       # Исходный код
└── src/lib/                   # Общие утилиты и компоненты
```

### Языковые паттерны промптов (из оригинала Geoff)
- "study" (не "read" или "look at")
- "don't assume not implemented"
- "using parallel subagents" / "up to N subagents"
- "only 1 subagent for build/tests"
- "Ultrathink"
- "capture the why"

## Продвинутые расширения

### Acceptance-Driven Backpressure
Из acceptance criteria в спеках выводятся конкретные требования к тестам. Предотвращает "жульничество" — нельзя считать задачу выполненной без прохождения тестов, привязанных к критериям приёмки.

### Non-Deterministic Backpressure (LLM-as-Judge)
Для субъективных критериев (эстетика, тон, UX) используется LLM в роли судьи. Утилита `llm-review.ts` с бинарным pass/fail API. Поддержка мультимодальности: текст и скриншоты.

### Ralph-Friendly Work Branches
Скоупированный план при создании ветки (`plan-work` режим). Ralph строит из уже скоупированного плана без семантической фильтрации в рантайме.

### JTBD → Story Map → SLC Release
Переосмысление "topics of concern" как activities (глаголы в пользовательском пути). Построение User Story Map. Горизонтальные срезы = кандидаты на релизы. SLC (Simple/Lovable/Complete) вместо MVP.

### Reverse Engineering Brownfield Projects
Обратная инженерия существующей кодовой базы в спецификации. Двухфазный процесс: расследование с полным доступом к коду → написание спеков без деталей реализации.

## Главный вывод

Ценность playbook не в самом цикле (`while :; do cat PROMPT.md | claude ; done`), а в **инженерии среды вокруг цикла**: как формулировать спецификации, как структурировать промпты, как создавать backpressure, как управлять контекстным окном, и как настраивать поведение агента через наблюдение и итеративные корректировки.

Формула: **"Let Ralph Ralph" + детерминированная настройка + backpressure через тесты = масштабируемая автономная разработка.**
